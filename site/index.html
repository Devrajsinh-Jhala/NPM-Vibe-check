Pre-execution safety for npm packages

A practical safety check before running npx.

npx-vibe inspects npm packages before they execute. It verifies package metadata and tarballs, reviews install-time behavior, and shows source-backed findings in a terminal report developers can act on.

Designed like a developer tool

Fast defaults, readable evidence, clear boundaries.

The goal is not to replace audits or sandboxing. The goal is to add a useful checkpoint before a package reaches install scripts or executable code.

Heuristic by default

Runs without API keys and ignores ambient provider credentials unless AI is explicitly requested.

Source evidence

Findings point to files, lifecycle hooks, and line-level excerpts instead of vague warnings.

Package context

Shows age, downloads, maintainers, publisher, repository, and install-hook metadata.

Review memory

Recognizes unchanged package integrity and helps compare what changed between versions.

Workflow

One command, three checks.

npx-vibe keeps the flow close to how developers already work: run a command, read the evidence, decide whether to continue.

  1. 01

    Resolve and verify

    Fetch npm metadata, resolve the exact package version, and verify the downloaded tarball.

  2. 02

    Inspect install behavior

    Review lifecycle scripts and selected files for shell, network, secrets, and obfuscation signals.

  3. 03

    Print a decision report

    Return Proceed, Caution, or Block with the concrete evidence behind the recommendation.

Terminal output

Reports meant to be read quickly.

The CLI output is intentionally plain: package identity, context, findings, evidence, and action.

Optional AI

Model review is explicit, not automatic.

Online or local AI can help interpret suspicious install scripts, but the deterministic scan remains the baseline. Provider and model names are printed in the output.

Local

Use Ollama when you want package review to stay on your machine.

Online

Use provider-specific keys for OpenAI, Gemini, Anthropic, OpenRouter, Groq, and more.

Profiles

Choose fast, balanced, or strong model profiles without memorizing model names.

Install

Start with a read-only check.

The default command inspects the package before execution. Use it when a package is new, unfamiliar, or has install-time behavior you want to understand.

Check a package npx npx-vibe --check esbuild
Install globally npm install -g npx-vibe
Optional Gemini review npx npx-vibe --ai online --provider gemini esbuild
Security boundary

npx-vibe is a pre-execution scanner. It is not a sandbox, antivirus product, or formal audit. It helps surface risky signals before execution, but it cannot prove a package is safe.

Built for developers who want fast tools with inspectable decisions.
