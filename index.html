npm supply-chain safety, before execution

Vibe check packages before you run them.

Replace a risky npx obscure-package with npx npx-vibe obscure-package. It inspects registry signals, install hooks, tarballs, package metadata, and optional AI review before letting code execute.

Use it like npx, but with a safety pause.

npx npx-vibe cowsay -- hello
npm install -g npx-vibe
npx-vibe --check obscure-package
npx npx-vibe --json suspicious-package

The fast path is deterministic and does not require an AI key. If a package trips risk signals, npx-vibe can escalate to local Ollama or an online AI provider.

Registry heuristics

Checks age, weekly downloads, version publish date, maintainers, license, npm update time, and repository metadata.

Tarball inspection

Downloads the package tarball without executing code, verifies integrity, and inspects install scripts and suspicious files.

Readable verdicts

Prints a terminal dashboard with Proceed, Caution, or Block, plus the evidence behind the call.

How it works

  1. Resolve the exact npm package version.
  2. Fetch registry and download metadata.
  3. Download and inspect the tarball safely.
  4. Run deterministic static heuristics.
  5. Optionally ask Ollama or an online model for review.
  6. Execute only after the verdict policy allows it.

Bring your own AI key, or keep it local.

Online mode supports OpenAI-compatible APIs, OpenAI, Anthropic, Gemini, OpenRouter, Groq, and Together AI. Local mode supports Ollama. 

OPENAI_API_KEY=... npx npx-vibe --ai online pkg
ANTHROPIC_API_KEY=... npx npx-vibe --ai online pkg
npx npx-vibe --ai ollama --ollama-model qwen2.5-coder pkg

Privacy boundary

npx-vibe never sends your local project files, npm tokens, shell history, or environment variables to a model. AI review receives only bounded package metadata, deterministic findings, install scripts, and selected files from the npm tarball.

Built for developers who like speed, but not surprise install scripts.
