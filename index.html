the safety pause

A fast checkpoint for the packages you do not fully know yet.

The default path is deterministic and does not need an API key. If a package has risky signals, npx-vibe shows the exact reason with enough context for a human decision.

01

Resolve and verify

Fetch registry metadata, resolve the exact version, download the tarball without execution, and verify npm integrity.

02

Inspect install behavior

Follow lifecycle scripts and selected files for shell, network, secrets, obfuscation, persistence, and native execution signals.

03

Compare what changed

Use local integrity-keyed memory to recognize unchanged tarballs or highlight file, hook, and finding deltas across versions.

one command

Start with a read-only check.

By default, npx-vibe runs heuristic analysis only. It ignores ambient provider keys, never executes package install scripts during inspection, and only uses AI when explicitly requested.

Check a package without installing it 
npx npx-vibe --check esbuild
Install globally 
npm install -g npx-vibe
Add online AI 
npx npx-vibe --ai online --provider gemini esbuild
List model profiles 
npx npx-vibe --models
Evidence-first

Repeated use

When a version changes, npx-vibe can compare selected file hashes, lifecycle script changes, and finding deltas without storing source code or secrets.

optional AI

Use a local model or bring your own provider.

Keep the default deterministic, use Ollama locally, or choose online model profiles for fast, balanced, or strong review. The exact provider and model are printed in the scan output.

honest boundary

A checkpoint, not a magic shield.

npx-vibe is not a sandbox, antivirus product, or formal audit. It inspects bounded npm package content before execution and can miss runtime-only, dependency, native-code, or remote-response behavior. A Proceed verdict is useful context, not proof of safety.

privacy boundary

Your workspace is not the prompt.

Online AI review receives bounded registry metadata, deterministic findings, and selected files from the downloaded npm tarball. It does not receive your project files, environment values, npm tokens, API-key values, or shell history.
